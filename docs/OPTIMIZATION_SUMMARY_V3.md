# 时间轴图v3交替上下布局优化总结

##  优化概述

本次优化针对**上下行内容块严重重叠**的问题，从v2版本升级到v3版本，采用**交替上下布局**策略彻底解决重叠问题。

---

## 🎯 核心问题

### v2版本存在的问题

在v2版本中，所有行的内容块都显示在时间轴的**上方**，导致：

1. **下一行覆盖上一行**：第2行的内容块会覆盖第1行的时间标签
2. **视觉混乱**：多行内容块堆叠在一起，难以阅读
3. **信息丢失**：重要的时间信息被遮挡

**示例场景**（6行布局）：
```
第1行 [09:00]─[09:05]─...─[09:30]    ← 内容块在上方
                         │
                         ╲ 
                         │
第2行 [09:40]─[09:50]─...─[10:45]    ← 内容块也在上方 ❌ 覆盖第1行时间！
                         │
                         ╲
                         │
第3行 [11:00]─[11:15]─...─[12:15]    ← 内容块也在上方 ❌ 覆盖第2行时间！
```

---

## ✅ v3解决方案

### 交替上下布局策略

采用**奇偶行交替**的布局方式：

- **第1、3、5行**（偶数索引）：内容块显示在时间轴**上方** ↑
- **第2、4、6行**（奇数索引）：内容块显示在时间轴**下方** ↓

**优化后的布局**：
```
第1行 [09:00]─[09:05]─...─[09:30]    ← 内容块在上方 ↑
                         │
                         ╲ 
                         │
第2行 [09:40]─[09:50]─...─[10:45]    ← 内容块在下方 ↓ ✅ 不覆盖！
                         │
                         ╲
                         │
第3行 [11:00]─[11:15]─...─[12:15]    ← 内容块在上方 ↑ ✅ 不覆盖！
```

   - 解决：交替布局确保所有时间标签可见

## 🔍 布局原理详解

### 行索引计算

```python
# 节点全局索引 gi (0, 1, 2, ..., 36)
# 每行最多6个节点 (MAX_PER_ROW = 6)

current_row_idx = gi // MAX_PER_ROW

# 示例：
# gi = 0-5  → row_idx = 0  → is_top = True  (上方)
# gi = 6-11 → row_idx = 1  → is_top = False (下方)
# gi = 12-17→ row_idx = 2  → is_top = True  (上方)
# gi = 18-23→ row_idx = 3  → is_top = False (下方)
# ...
```

### 交替逻辑

```python
# 偶数行（0, 2, 4...）：内容块在上方
if current_row_idx % 2 == 0:
    is_top = True
    label_y = timeline_y + text_offset
    label_va = 'bottom'
else:
    # 奇数行（1, 3, 5...）：内容块在下方
    is_top = False
    label_y = timeline_y - text_offset
    label_va = 'top'
```
